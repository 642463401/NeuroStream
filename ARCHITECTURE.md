# 🧠 项目代号：NeuroStream (双线程持续学习架构)

## 1. 核心设计哲学
本项目旨在构建一个**解耦的持续学习系统**，而非传统的静态推理模型。
- **核心目标**：实现“边推理边学习”，解决灾难性遗忘问题，支持多模态输入输出。
- **关键隐喻**：模拟生物大脑的“皮层（快思考/推理）”与“海马体（慢思考/巩固）”机制。
- **记忆原则**：**上下文只是原材料，记忆池才是知识库**。所有上下文必须经过“提炼”才能进入记忆池。

## 2. 系统架构概览
系统由两个独立运行的线程（进程）组成，通过共享内存（记忆池）进行交互。

### 🧵 线程 A：推理线程
- **职责**：实时处理多模态输入，生成输出。
- **特性**：低延迟，只读模式（读取记忆池，不修改权重）。
- **工作流**：
    1. 接收多模态输入（图像/文本/音频）。
    2. **统一编码**：将输入转化为公共向量空间的概念向量。
    3. **唤醒机制**：以当前概念为 Query，从记忆池中检索相关记忆（Key-Value Retrieval）。
    4. **融合推理**：结合短期上下文 + 唤醒的长期记忆 -> 生成输出。
    5. **写入短期缓冲**：将当前交互存入“短期记忆缓冲区”。

### 🧵 线程 B：计算/学习线程
- **职责**：后台异步运行，负责记忆的“提炼”与权重的“优化”。
- **特性**：高计算量，写模式。
- **工作流**：
    1. **监控缓冲区**：读取短期记忆缓冲区的数据。
    2. **时间积分**：对重复出现的模式进行积分计算（$S_t = S_{t-1} \cdot decay + I_{new}$）。
    3. **剪枝与固化**：
        - 低强度/低关联度数据 -> **丢弃**。
        - 高强度/高关联度数据 -> **写入长期记忆池**。
    4. **影子权重更新**：基于固化记忆计算梯度，平滑更新推理线程的影子权重。

## 3. 记忆池设计
记忆池是系统的核心，必须支持多模态向量存储。
